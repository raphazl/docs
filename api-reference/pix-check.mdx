## Visão geral

Consulte o status atual de uma cobrança criada via `/pix/create`. Ideal para polling no frontend enquanto o cliente ainda não pagou.

<Info>
  Para receber a confirmação automaticamente sem precisar fazer polling, configure um [webhook](/api-reference/webhooks) na criação da cobrança.
</Info>

---

## Request

```bash
curl -X GET "https://api.gekkopay.com.br/pix/check?id=683fb19a5d5de60d9bbc4e3f" \
  -H "Authorization: Bearer sk_live_..."
```

### Query parameters

<ParamField query="id" type="string" required>
  ID da transação retornado no `/pix/create`.
</ParamField>

## Status possíveis

| Status | Descrição |
| :-- | :-- |
| `PENDING` | Aguardando pagamento. O QR Code ainda é válido. |
| `PAID` | Pagamento confirmado. O valor já foi creditado na conta do vendedor. |
| `EXPIRED` | O tempo limite de pagamento foi excedido. Crie uma nova cobrança. |
| `FAILED` | Erro no processamento. Tente novamente. |

## Exemplo de polling

<Steps>
  <Step title="Crie a cobrança">
    Faça um `POST /pix/create` e salve o `data.id` retornado.
  </Step>
  <Step title="Exiba o QR Code">
    Renderize o `qrCode` e o `pixCode` no checkout para o cliente.
  </Step>
  <Step title="Faça polling do status">
    Chame `GET /pix/check?id=...` a cada **3–5 segundos**.
  </Step>
  <Step title="Detecte o pagamento">
    Quando `status` mudar para `PAID`, redirecione o cliente para a página de sucesso.
  </Step>
</Steps>

<Warning>
  Não faça polling com intervalo menor que 2 segundos. Requisições excessivas podem ser limitadas pelo rate limiter.
</Warning>
